# F.18 Name Card: Executor Role

## 1. Twin-Labels
- **Technical ID**: `executor-role`
- **Plain-English**: "Executor Role"

## 2. Minimal Definitional Statement (MDS)
> The role that carries out a handoff by applying changes, running checks, and reporting results with evidence.

## 3. Context of Meaning
Defined within the **AgentWorkflow** bounded context.

## 4. Rationale
Keeps execution accountable and distinct from planning, enabling precise Work records for the two-agent loop.
